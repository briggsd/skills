# Capturing a primary source after a commentary capture

When a user first captures commentary on a source, then says to capture the original/source article “on top of that,” treat the source as a separate capture and explicitly cross-link both notes.

Recommended flow:

1. Extract the primary source normally and decide tier from the user’s wording. If the commentary was upgraded to Tier 2 and the source is requested as supporting source material, default the primary source to Tier 2 unless the user asks for quick capture only.
2. Create a distinct capture for the primary source using its own title/author/date metadata.
3. In the source capture, add a `Related capture` or `Connections` entry pointing to the commentary capture and clarify that the commentary is interpretive while this note is the original engineering/source material.
4. In the commentary capture, add a reciprocal link near `Connections` to the primary source capture.
5. Preserve the distinction in summaries: commentary captures can include framing/opinion; primary-source captures should emphasize implementation details, measured claims, and direct limitations from the source.
6. If both captures point to the same topic docs, suggest distilling them together to avoid duplicating near-identical insights in topic docs.

This pattern prevents treating a reaction video/blog as the authoritative source while still keeping both useful: the commentary captures interpretation; the source captures implementation evidence.
